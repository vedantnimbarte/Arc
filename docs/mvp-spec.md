# MVP Spec (Phase 1)

Distilled from the full product spec into the **smallest thing that proves the model**.

## In

- One window, dark theme, monospace.
- Tab bar with new-tab button (Ctrl+T) and close-tab.
- Multiple terminal tabs, each backed by a real PTY (cmd.exe on Windows, $SHELL elsewhere).
- xterm.js with: 256-color, mouse, resize, scrollback 10k, web-links addon.
- AI chat panel docked on the right. Sends to a **stub** provider that echoes the message.
- Status bar showing runtime (`tauri`/`web-only`) and active tab id.
- Tabs survive within a session; not persisted across launches yet.

## Out (Phase 2+)

- Editor tabs (CodeMirror integration).
- Real AI providers (OpenAI, Anthropic, Ollama).
- Workspace/session persistence.
- Agent runtime, background agents.
- Memory / vector search.
- Plugin system.
- Collaboration / cloud sync.
- Command blocks (Warp-style).
- Git integration UI.

## Success criteria

- `pnpm tauri:dev` launches a window in under ~5 seconds (warm cargo cache).
- A terminal tab spawns `cmd.exe` (Windows) and renders `cd`, `dir`, `git status`, etc. without lag.
- Closing a tab kills the underlying child process.
- Resizing the window resizes the shell (verify with `mode con` on Windows or `tput lines`).
- The chat panel streams the stub reply word-by-word — proves the UI streaming path.

## Known limitations baked into the MVP

- PTY data is JSON-encoded byte arrays — fine for shell text, slow for `cat big.log`. (ADR-006)
- Tab state is in-memory only.
- No telemetry, no auto-update, no signing.
- Icons are placeholders generated by PowerShell's System.Drawing.
